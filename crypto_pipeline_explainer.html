📂 Data Loading

The dataset (OHLCV — Open, High, Low, Close, Volume) is loaded from the CSV file into a Pandas DataFrame. The Date column is parsed so Python understands it as actual dates, not plain text.

10 df = pd.read_csv('dataset.csv', parse_dates=['Date']) ← load CSV, auto-parse Date column

🔍 Exploratory Data Analysis (EDA)

Before building anything, we examine the data — its shape, data types, missing values, and statistical summary. This reveals what needs to be cleaned and what patterns exist.

15 print(df.shape) ← (rows, columns) — size of dataset

04
🧹 Data Preprocessing & Cleaning

Raw market data often contains missing values (trading holidays, API gaps). Here we handle NaNs and ensure numeric columns are in the right format. We also remove any duplicates.

22 df.dropna(inplace=True) ← drop rows with any missing value
23 df.drop_duplicates(inplace=True) ← remove duplicate records
24 cols = ['Open', 'High', 'Low', 'Close', 'Volume'] ← define numeric columns
25 df[cols] = df[cols].apply(pd.to_numeric, errors='coerce') ← force numeric, bad values → NaN
26 df.fillna(method='ffill', inplace=True) ← forward-fill remaining NaNs
05
⚙️ Feature Engineering

Raw OHLCV columns alone are weak predictors. We engineer technical indicators — moving averages, RSI, Bollinger Bands, daily return — to give models meaningful signal about price momentum and volatility patterns.

28 df['Daily_Return'] = df['Close'].pct_change() ← % change day-over-day
29 df['MA_7'] = df['Close'].rolling(7).mean() ← 7-day moving average
30 df['MA_30'] = df['Close'].rolling(30).mean() ← 30-day moving average
31 df['Volatility_7'] = df['Daily_Return'].rolling(7).std() ← 7-day rolling std (volatility proxy)
32 df['High_Low_Pct'] = (df['High'] - df['Low']) / df['Close'] ← intraday range as % of close
33 df['Upper_BB'] = df['MA_30'] + 2 * df['Close'].rolling(30).std() ← upper Bollinger Band
34 df['Lower_BB'] = df['MA_30'] - 2 * df['Close'].rolling(30).std() ← lower Bollinger Band
35 df.dropna(inplace=True) ← rolling windows create NaNs at start
06
🎯 Target Variable Definition

Volatility is defined as the rolling standard deviation of daily returns over a 30-day window. This is the value the model will learn to predict. All other engineered columns become the input features (X).

38 df['Target_Volatility'] = df['Daily_Return'].rolling(30).std() ← 30-day realised volatility (y)
39 df.dropna(inplace=True) ← drop rows where target is NaN
41 features = ['Open', 'High', 'Low', 'Close', 'Volume',
42 'MA_7', 'MA_30', 'High_Low_Pct', 'Upper_BB', 'Lower_BB'] ← all 10 input features (X)
43 X = df[features] ← feature matrix
44 y = df['Target_Volatility'] ← label vector
07
✂️ Train / Test Split & Feature Scaling

We split the data 80/20 — 80% for training, 20% for testing. shuffle=False is critical here: time-series data must stay in order, otherwise future data would leak into training. StandardScaler then normalises all features to mean=0, std=1 so no single feature dominates.

47 X_train, X_test, y_train, y_test = train_test_split(
48 X, y, test_size=0.2, shuffle=False) ← 80/20 split, NO shuffle (time-series!)
50 scaler = StandardScaler() ← initialise scaler
51 X_train = scaler.fit_transform(X_train) ← fit on train, transform train
52 X_test = scaler.transform(X_test) ← ONLY transform test (no re-fitting!)
08
🤖 Model Training — Multiple Algorithms

Several ML models are trained and compared so the best performer can be selected. Each model receives the same scaled training data.

Linear Regression
Baseline
Fits a straight line through the data. Simple and fast — useful as a performance baseline.
Low Complexity
Random Forest
Ensemble
Builds hundreds of decision trees and averages their predictions. Robust to outliers and overfitting.
Medium Complexity
XGBoost
Gradient Boosting
Iteratively corrects errors of previous trees. Typically best performance on tabular financial data.
High Performance
LSTM
Deep Learning
Long Short-Term Memory network — captures temporal dependencies in sequential price data.
Sequential

55 from sklearn.linear_model import LinearRegression
56 from sklearn.ensemble import RandomForestRegressor
57 from xgboost import XGBRegressor
59 lr = LinearRegression().fit(X_train, y_train) ← train linear model
60 rf = RandomForestRegressor(n_estimators=100).fit(X_train, y_train) ← 100 decision trees
61 xgb = XGBRegressor(n_estimators=200).fit(X_train, y_train) ← 200 boosting rounds
09
📊 Model Evaluation & Comparison

Each model's predictions are compared against the actual test volatility using standard regression metrics. Lower RMSE and higher R² = better model.

METRIC 1
RMSE
Root Mean Squared Error
← penalises large errors
METRIC 2
MAE
Mean Absolute Error
← average prediction gap
METRIC 3
R²
R-Squared Score
← 1.0 = perfect fit

65 def evaluate(model, name): ← reusable eval function
66 preds = model.predict(X_test) ← generate predictions on test set
67 rmse = np.sqrt(mean_squared_error(y_test, preds)) ← root mean squared error
68 r2 = r2_score(y_test, preds) ← variance explained by model
69 print(f"{name} → RMSE:{rmse:.4f} R²:{r2:.4f}") ← print formatted results
71 for m, n in [(lr,'Linear'),(rf,'RandomForest'),(xgb,'XGBoost')]:
72 evaluate(m, n) ← evaluate all 3 models
